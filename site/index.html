核心差异

别人的文档会悄悄烂掉
Yoke 的文档会主动报警

活文档最大的问题不是写得不好，而是写完之后过时——而且是悄悄过时，你不知道哪句话已经失效。Yoke 用内容哈希把每条文档钉在它描述的那段代码上。

⚠ 过时检测

改一行代码，谁过时了一目了然

每条 learning、每个组件规约、每条 trace 都锚定到具体的函数或类。代码一改，锚点的哈希就变，对应文档立刻翻成 stale——在代码移动的那一刻，而不是等某个 agent 后来碰巧发现文档不对。

# coder 改完代码，收尾自动扫描 $ anchor scan Scanned 12 anchors — fresh 8 · stale 3 · missing 1 A-e428 fresh → stale (learning:feat-id-format @ search.py generate_feat_id)
✓ 局部再生

只复核过时的那几条，省 token

update-map 不再扫全部文档，只针对报红的锚点复核、改文档、重定基线。审计阶段还会把未处理的过时文档计入扣分——文档新鲜度第一次有了机器可检的硬闸门，不靠人记得去看。

# 改好文档后重定基线 $ anchor verify --anchor-id A-e428 verified A-e428 — re-baselined to 1fc3945f (fresh) # 全部回到同步 $ anchor report All anchored docs are in sync.

工作原理

一条流水线，自适应调度

两个需求入口共享同一条流水线。pipeline 根据任务复杂度（PATCH / FEATURE / REFACTOR）自动决定走哪些阶段——小改动跳过 spec，新功能走完整 TDD。

propose (PM) ┐
            ├─▶ pipeline → architect → qa → coder → audit → update-map → ship

                     guard               anchor + learn
